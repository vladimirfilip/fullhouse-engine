# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Context

This is a fork of the Fullhouse poker bot competition engine. The competition runs 1–5 June 2026. Work happens in `bots/vlad/` — a No-Limit Texas Hold'em bot competing in a Swiss-system tournament. All other engine files (`engine/`, `sandbox/`, `demo.py`) are frozen and should not be modified.

## Commands

```bash
# Install sandbox-runtime deps (eval7 requires Cython<3 + no-build-isolation)
make install

# Install training deps (PyTorch CPU + pybind11)
make install-train

# Install training deps with CUDA 12.1
make install-train-gpu

# One-shot bootstrap for a fresh Linux training VM (Python 3.10 venv + GPU torch + C++ build)
make install-vm

# Build the C++ data-generation extension (deep_cfr_gen pybind11 module)
make build-cpp

# Run tests
make test
# or directly:
python -m pytest tests/ -q

# Validate bot before submission
make validate BOT=bots/vlad/bot.py

# Run interactive demo UI (localhost:5000)
make demo

# Train (full: 100 iters × 10,000 games)
make train

# Smoke-test training run (5 iters × 200 games)
make train-quick

# Clean compiled artifacts
make clean
make clean-cpp
```

## Architecture

### Engine (frozen)

**`engine/game.py`** — `PokerEngine` class: pure-Python NLHE rules for 2–9 players. Constants: `SMALL_BLIND=50`, `BIG_BLIND=100`, `STARTING_STACK=10_000`. Handles betting rounds, side pots, showdowns via eval7. Emits a structured event log per hand.

**`engine/tournament.py`** — Swiss-system pairing and standings. Ranking tiebreakers: cumulative chip delta → matches played → best match delta → bot_id.

**`sandbox/runner.py`** — Bot process executor. Loads `bot.py`, communicates via newline-delimited JSON on stdin/stdout. Enforces **2s per-action timeout** (threading-based for Windows compat) and **30s warmup timeout**.

**`sandbox/match.py`** — Orchestrates a full 400-hand match. Injects a rolling `match_action_log` (up to 200 entries) so bots can model opponents across hands. Returns chip deltas and hand history.

**`sandbox/validator.py`** — Static + dynamic pre-submission checks. Forbidden imports: `socket`, `requests`, `subprocess`, `multiprocessing`, `pickle`, `eval`, `exec`, `compile`.

### Bot Protocol

Each action, `runner.py` calls `decide(game_state: dict) -> dict` in the bot. The game state includes:

- `your_cards`: `["As", "Kh"]` — hole cards
- `community_cards`: board cards (empty preflop)
- `street`: `"preflop" | "flop" | "turn" | "river"`
- `pot`, `your_stack`, `amount_owed`, `can_check`, `current_bet`, `min_raise_to`
- `players`: list of opponent info (stack, state, bet_this_street)
- `action_log`: all actions this hand
- `match_action_log`: rolling cross-hand history (up to 200 entries)

Valid return values:
```python
{"action": "fold"}
{"action": "check"}
{"action": "call"}
{"action": "raise", "amount": <int>}  # total bet amount, >= min_raise_to
{"action": "all_in"}
```

### `bots/vlad/bot.py` — Current Implementation

Loads a pre-trained GTO strategy network at import time (`bots/vlad/data/gto_strategy.npz`). Falls back to Monte Carlo equity if the model is unavailable.

**GTO path:** `_build_feature_vector(gs)` → 308-element float32 vector → pure-numpy MLP forward pass (LeakyReLU activations) → softmax → pick highest-probability legal action. RNG seeded from game state for replay stability.

**Real-time EV blending:** `_realtime_search(gs, legal, gto_probs)` — 600-iteration MC equity + 1-level lookahead. Fold model: `fp = min(0.10 + 0.18 × (bet/pot), 0.70)`. Decision: 60% GTO network + 40% EV signal.

**MC fallback:** `monte_carlo_equity(...)` — shuffles deck, evaluates with eval7. Includes opponent profiling (maniac / calling station / nit detected from match action log) to adjust equity buffers.

**Feature vector layout** (308 floats, byte-for-byte identical across bot.py, features.cpp, config.py):

| Range | Content |
|---|---|
| [0:52] | Hole cards one-hot |
| [52:104] | Board cards one-hot |
| [104:110] | Hero position relative to dealer (one-hot, 6 seats) |
| [110:117] | Pot + per-seat stacks normalised by INITIAL_STACK |
| [117:135] | Per-seat folded/all-in/bet-this-street masks (3 × 6) |
| [135:139] | Street one-hot (preflop/flop/turn/river) |
| [139:145] | Pot odds, log-SPR, amount owed, raise count, hero bet, eff stack |
| [145:158] | Last aggressor: one-hot seat (6) + amount + relative position (6) |
| [158:163] | Board texture: flush-draw, monotone, paired, two-paired, connected |
| [163] | n_active players (normalised) |
| [164:308] | Action history: 24 slots × 6 floats (seat, 4-action one-hot, amount) |

### `deep_cfr/` — Python Training Orchestration

Offline PyTorch training that consumes data generated by the C++ extension. Produces `bots/vlad/data/gto_strategy.npz`.

**Key files:**

- **`config.py`** — all hyperparameters: `N_ACTIONS=9`, `INPUT_DIM=308`, `HIDDEN_DIM=512`, `N_LAYERS=4`, `K_ITERATIONS=100`, `GAMES_PER_ITER=10_000`, `BATCH_SIZE=4_096`, `REGRET_TRAIN_STEPS=5_000`, `STRATEGY_TRAIN_STEPS=15_000`, `N_WORKERS` (auto = CPU count).
- **`networks.py`** — `make_regret_net()` and `make_strategy_net()`. Both are 4-layer MLPs (512 hidden, LeakyReLU 0.01). Regret net: linear output (regrets can be negative). Strategy net: softmax output. Output dim = 9.
- **`export.py`** — `export_net(net, path)` serialises Linear weights/biases as numpy `.npz` (`layer0_w`, `layer0_b`, …, `n_layers`) for sandbox inference.
- **`train.py`** — Main loop. Loads the `deep_cfr_gen` C++ extension (pybind11). Each iteration: spawn N_WORKERS C++ threads for MCCFR traversals (writes to thread-safe C++ reservoir buffers) → train regret net (double-buffered prefetch) → checkpoint → every 5 iters train strategy snapshot. Final phase: 15K-step strategy net. Device: CUDA > DirectML > CPU.

**Training CLI:**

```bash
python -m deep_cfr.train            # full run
python -m deep_cfr.train --quick    # 5 iters × 200 games smoke test
python -m deep_cfr.train --iters 20 --games 5000 --workers 8
```

### `deep_cfr_cpp/` — C++ Data Generation

Built by `make build-cpp` into a pybind11 extension module `deep_cfr_gen`. Not used at runtime — only for offline training.

**Source files (`src/`):**

- **`config.hpp`** — mirror of `config.py` with C++ constants (exact numeric equivalents required for training/inference consistency).
- **`engine.hpp/.cpp`** — Poker game engine: `init()`, `start_hand()`, `apply_action()`, dealing, blind posting, street advances.
- **`features.hpp/.cpp`** — `build_feature_vector(state_dict)` → 308-element `FeatureVec`. Single-pass action log: finds last aggressor + fills 24-slot action history ring buffer. Board texture detection.
- **`hand_eval.hpp/.cpp`** — Hand strength evaluation.
- **`mccfr.hpp/.cpp`** — External Sampling MCCFR. Chance nodes: deal cards. Opponent nodes: sample one action, append `(feature_vec, strategy, t)` to strategy buffer. Traverser nodes: enumerate all actions, compute regrets, append `(feature_vec, regret_vec)` to regret buffer. Returns EV for traverser.
- **`network.hpp/.cpp`** — C++ MLP inference (`forward_single`) using Eigen. Reads weight matrices shipped from Python as numpy arrays.
- **`reservoir.hpp`** — Thread-safe reservoir sampling buffers.
- **`train.hpp/.cpp`** — C++ training loop helpers.
- **`binding.cpp`** — pybind11 bindings exposing the above to Python as `deep_cfr_gen`.
- **`third_party/miniz.h`** — single-header miniz for .npz I/O.

**Build dependencies:** CMake fetches Eigen 3.4.0 and pybind11 v2.13.6 automatically via FetchContent. Requires: C++17 toolchain, cmake ≥ 3.16.

**Abstract action space** (index → meaning):

| Index | Name | Description |
|---|---|---|
| 0 | FOLD | Fold |
| 1 | CHECK_CALL | Check or call |
| 2 | BET_027_POT | Bet/raise to 0.27× pot (off-grid) |
| 3 | BET_033_POT | Bet/raise to 0.33× pot |
| 4 | BET_HALF_POT | Bet/raise to 0.5× pot |
| 5 | BET_FULL_POT | Bet/raise to 1× pot |
| 6 | BET_172_POT | Bet/raise to 1.72× pot (off-grid) |
| 7 | BET_2X_POT | Bet/raise to 2× pot |
| 8 | ALL_IN | All-in |

The 0.27× and 1.72× sizes are deliberately off-grid to exploit standard GTO solver trees.

**Important constraints:** PyTorch is only needed for training (offline). The sandbox forbids subprocess/multiprocessing, so `train.py` must never be imported by `bot.py`. The numpy-only forward pass in `bot.py` is what runs in production. The 308-dim feature vector must remain byte-for-byte identical across `features.cpp`, `config.py`, and `bot.py` — any drift silently corrupts the model.

## Hardware Constraints (bots/vlad/)

All code in `bots/vlad/` must operate within these production sandbox limits:

| Resource | Limit |
|---|---|
| RAM | 768 MB |
| CPU | 0.5 cores |
| Action timeout | 2 seconds |
| Warmup timeout | 30 seconds |
| `/tmp` | 20 MB |
| `data/` directory | 200 MB (read-only) |
| Total submission | 250 MB |

Heavy resources (model weights, lookup tables) must be loaded once at module import time (warmup), not inside `decide()`. The filesystem is read-only at runtime — no file writes. No network access. No subprocess/threading spawning inside bots.

## Submission Format

A valid submission is either:
- `bots/vlad/bot.py` (single file, ≤ 5 MB)
- `bots/vlad/bot.zip` containing `bot.py` at root + optional `data/` subdirectory (≤ 200 MB)

Data files in `data/` are mounted read-only at `/data` inside the sandbox.
